# docs/

This folder is for stable supporting documentation only.

Examples:
- operator notes
- hardware notes
- pin maps
- environment setup notes
- recovery procedures
- workflow explanations that should not live in the top-level state files

Do not use this folder for transient run logs or temporary scratch notes.
Operational status should stay in:
- `TODO.md`
- `PROJECT_STATE.md`
- `RUNBOOK.md`
- `LESSONS.md`
